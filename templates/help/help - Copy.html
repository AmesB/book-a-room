Book a Room Administration

Help!

The Book a Room plug-in comes in two parts. 

The Meeting Room plugin manages most of the heavy lifting. It manages staff and public meeting room reservations, settings and administrative functions.

The Event Calendar plugin displays several different things. It shows the public event calendar and gives users detailed event information. It also provides registration forms and a public search function. 

The Event Calendar plug-in can be installed in a separate Wordpress installation. This allows you to offer your events on a different subdomain if needed. It can also run side-by-side with the Meeting Room plug-in on the same site.

Main Settings

Use the Main Settings menu to configure the Meeting Room System. There is a key on the page itself that explains each option in detail.

Things to remember

  • Most time-related settings rely on the Base increment in minutes value. This is the number of minutes in the smallest reservable increment. If you want to reserve rooms in half hour increments then this value would be 30.
  • The Setup and Cleanup increments are the number of increments before (setup) and after (cleanup) each meeting. If you use both, there will be an hour between each meeting. It's best to choose one or the other.
  • For the URL for reservations page, check your permalink structure at Settings > Permalinks.

    If your permalinks are set to "Default", then just use the ID of the page that you have created for your Meeting Room Sign up page.
    If you created a page and the end of the page is: ?page_id=23, then you would just put 23 as your setting.

    If your permalinks are set to anything else, look at the link and include everything after your site's domain name.
    For instance, if your page's URL is: http://lookatmysitename.com/meeting-rooms/, then you would enter "/meeting-rooms" (sans quotes). Include the first slash, but not any trailing slash.
  • The Days buffer and Allowed buffer will give you a window where public users are allowed to reserve. If you or your staff are logged in to your site, these are ignored so staff can schedule any date.
  • The Default email for reservations is just the email that appears in the email box on the daily meeting room report (Daily Schedules > Daily Meetings).
  • The screen width defaults to Responsive/Wide. This may look a little odd if your theme is thin. You can force it into a thinner width by choosing narrow from the Screen Width drop down.

Shortcode

On the page you would like to show the meeting room signup and form, put the shortcode [meetingRooms] on the page.

Amenities

There are two types of Amenities: reservable and non-reservable. When you create an amenity, you only need to add a name and check or uncheck "Reservable?"

A reservable amenity is one that is available to reserve for a room but isn't a permanent fixture. Examples include TVs, DVD players, Whiteboards and projectors.

A non-reservable amenity is something you would like to let your public know is available but that isn't reservable. Examples could include a refrigerator, sink, projector screen or PA system.

Configuring your amenities first means that you can add them to each room as you create those rooms. If you create amenities last, you will have to edit your rooms to include all the amenities.

Branches

Branches are physical locations that contain rooms (and room containers). Each branch has a name, address, map link and image that will display on the main meeting room page. The option "Has 'No Location'" will add another option to each branch that allows staff to schedule events with no dedicated room. This could be for large events that aren't in a specific room or are outside on library grounds. The hours options here will also limit what times the branch and rooms are open for reservations. The times must be formatted in 12 hour time with both hour and minutes (i.e. 10:00, 5:00). Leaving both open and close times blank will mean the branch is closed on that day.

Rooms

Each room is a representation of a physical space in the branch. Rooms are not reservable themselves, but just describe an area. You only need to enter a name and branch for each room.

Room Containers

Room containers are virtual representations of reservable spaces. A room container can contain one or more rooms. A room container containing 2 rooms (like "Room A and B") is a reservable space that lets the system know that both room A and room B are reserved at a specific time. If you have three separate containers, "Room A", "Room B" and "Room A & B" which contain one or both of two rooms, then when a person reserves "Room A & B", the system knows that both rooms are in use at that time and neither Room A, B or a combination of both will be available to reserve.

The Max Occupancy value is the maximum amount of people that a room container can hold.

The Public? option will allow you to make the room container publicly available. Otherwise, a room container is only available to staff for reservations. This is great for organizing and tracking art galleries and display cases, allowing staff to reserve chunks of time for a display without letting the public access or reserve the space.

The Hide on daily? option works with the Public? option to hide a container from the daily meeting room list. If you have a room that is reservable to staff that you don't want showing up on the daily events, you would check this.

The Event Calendar

Main Settings

The main settings page for the Event Calendar is simply the connection information for the Wordpress installation that your Meeting Room System is running on. This allows the Event Calendar connect to your Meeting Room System information and display it on the public side. It's located in Settings > Event Settings.

Shortcode

Once the plugin is configured, you only have to add a small shortcode to the page you wish your run the plugin. The shortcode for the Event Calendar is [showCalendar].
